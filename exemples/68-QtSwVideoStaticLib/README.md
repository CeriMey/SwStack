# Qt Static Video Widget Example

This example packages the SwStack video widget behind a small Qt `QWidget` API
and builds it as a static library.  The demo application only compiles its own
`main.cpp` and links the generated `.a`.

Build with Qt MinGW:

```powershell
C:\Qt\6.9.3\mingw_64\bin\qmake.exe .\SwQtVideoStaticExample.pro
C:\Qt\Tools\mingw1310_64\bin\mingw32-make.exe
```

The demo app links only against:

```qmake
INCLUDEPATH += ../lib
LIBS += ../prebuilt/mingw/libSwQtVideoPlayer.a
```

plus the Windows system libraries required by the static SwStack media code.
The `prebuilt/` output is generated by qmake and is intentionally not tracked.

Run:

```powershell
.\demo\release\QtStaticVideoPlayer.exe D:\streams\camera.sdp
```

Supported URLs are the same as `SwMediaSourceFactory`, including `rtsp://`,
`rtp://`, `udp://`, and direct `.sdp` files.

To copy this into another Qt/qmake project after building the static library, keep:

- `lib/SwQtVideoPlayerWidget.h`
- `prebuilt/mingw/libSwQtVideoPlayer.a`
- the `LIBS` block from `demo/QtStaticVideoPlayer.pro`

Or assemble the standalone SDK folder after building/deploying:

- `standalone-mingw/include`
- `standalone-mingw/lib/mingw`
- `standalone-mingw/bin`
- `standalone-mingw/example`

The standalone example can be rebuilt outside this repository with:

```powershell
.\standalone-mingw\build.ps1
```

The demo toolbar has an editable URL field. Press `Enter`, `Open`, or
`Reconnect` after changing the URL to reopen the stream cleanly.
